目的・学べること

画面デザインのモックアップ（スクリーンショット）をClaude Codeに渡し、
本格的なAngularコンポーネントを生成する体験をします。

習得スキル

  • 画像からUIコードを生成するワークフロー
  • デザイン再現度を高めるプロンプトの書き方
  • 生成コードのコンポーネント分割指示
  • 複数画面にわたる共通コンポーネントの再利用

到達イメージ

  • 6画面の金融ダッシュボードを構築
  • 共通レイアウト・UIコンポーネントの分離
  • 画面遷移（Angular Router）の実装
  • 反復的な改善サイクルの体得

このハンズオンのポイント

HTMLソースコードではなく、画像（見た目の情報のみ）からUIを再現する

やること

  • スクリーンショット（PNG）をClaude Codeに渡す
  • 画像のデザインを言語化してプロンプトに含める
  • 生成されたコードをモックアップと比較・改善

やらないこと

  • HTMLソースコードのコピー&ペースト
  • 既存のコードを渡して変換させる
  • 100%完璧な再現を追求する

技術スタック

フレームワーク

Angular 17+

  • Standalone Components
  • TypeScript strict mode
  • Angular Router

スタイリング

Tailwind CSS 3.4

Step 1: ダッシュボード画面の生成

なぜ最も複雑な画面から始めるのか？

  • サイドバー・ヘッダー等の共通レイアウトが定義される
  • Card, Badge, Table等の基本UIコンポーネントが生まれる
  • カラー・フォント・間隔のデザイン基調がコードに反映

プロンプト例

プロンプト ../screenshots/01-dashboard.png を見てください。 これは金融ダッシュボードアプリのメイン画面の デザインモックアップです。 この画像のデザインをAngular standalone components + TypeScript + Tailwind CSSで実装してください。 実装要件: - Angular Router でデフォルトルートに設定 - サイドバーとヘッダーは app.component.ts に配置 - KPIカードは再利用可能な standalone component - チャートは Chart.js + ng2-charts を使用 - デザインのカラー、間隔を正確に再現 - 日本語テキストはモックアップと同じ内容

Step 1: チェックポイント

生成されたコードを以下の観点で検証してください:

レイアウト・デザイン

  • モックアップとレイアウトが一致しているか
  • サイドバーのナビゲーション項目が全て揃っているか
  • KPIカードの値・アイコン・色が正しいか

機能・データ

  • チャートが表示されているか（描画されることが重要）
  • テーブルのカラム構成とバッジが再現されているか
  • レスポンシブ対応がある程度効いているか
ポイント: 100%の再現は不要。80-90%の精度が出ていればOK。差分は後から修正指示を出す。

Step 2: コンポーネント分割

Step 1 の生成コードを再利用可能なコンポーネントに分離します。

プロンプト 生成されたダッシュボード画面のコードをコンポーネントに分割してください。 分割方針: - components/layout/sidebar.component.ts - サイドバーナビゲーション - components/layout/header.component.ts - ヘッダー - components/ui/kpi-card.component.ts - KPI表示カード（汎用） - components/ui/status-badge.component.ts - ステータスバッジ - components/ui/data-table.component.ts - データテーブル - components/charts/asset-line-chart.component.ts - 折れ線チャート - components/charts/sector-pie-chart.component.ts - 円グラフ 各コンポーネントは standalone: true で、@Input() に適切な型定義を持つこと。
分割後もブラウザ表示が崩れないことを必ず確認してください。

Step 3: 残り5画面の順次生成

共通コンポーネントを再利用しながら、各画面を順番に生成します。

3-1. ポートフォリオ管理

フィルターバー、損益率の色分け、ページネーション

→ /portfolio

3-2. 取引入力

注文フォーム（Reactive Forms）、気配値ボード、確認モーダル

→ /trade

3-3. リスク分析

VaR表示、リスクゲージ(SVG)、相関マトリクス、ストレステスト

→ /risk

3-4. 顧客詳細

プロフィールカード、タブ切り替え、KYC情報、確認書類

→ /clients/:id

3-5. コンプライアンス

アラートカード、不審取引検出、プログレスバー、監査ログ

→ /compliance

各画面で 既存の共通コンポーネント（Sidebar, Header, KpiCard, StatusBadge等）の再利用 を明示的にプロンプトに含める

Step 3: 各画面のプロンプト例

ポートフォリオ管理画面

プロンプト ../screenshots/02-portfolio.png を見てください。 ポートフォリオ管理画面のモックアップです。 既存の共通コンポーネント （SidebarComponent, HeaderComponent, DataTableComponent, StatusBadgeComponent）を 再利用して実装してください。 新規で必要なコンポーネント: - フィルターバー（ドロップダウン） - 損益率の色分け（＋緑、−赤） - ページネーション ルーティング: /portfolio を追加

リスク分析画面

プロンプト ../screenshots/04-risk-analysis.png を見てください。 リスク分析画面のモックアップです。 VaR表示、リスク指標ゲージ、相関マトリクス、 ストレステスト結果を実装してください。 新規コンポーネント: - VaR表示パネル - リスク指標ゲージ（SVGメーター型） - 相関マトリクスヒートマップ - シナリオ分析テーブル ルーティング: /risk を追加
共通パターン: スクリーンショットを渡す → 既存コンポーネントの再利用を指示 → 新規コンポーネントを明示 → ルーティング追加

Step 4: 画面遷移・ルーティング

プロンプト すべての画面が完成しました。 以下の画面遷移を実装してください: 1. サイドバーの各リンクを Angular の routerLink で接続 2. 現在のページに応じて routerLinkActive で アクティブ状態を切り替え 3. ダッシュボードの「すべて表示」を 取引画面に接続 4. 顧客管理画面はダッシュボードから 直接リンク

チェックポイント

  • すべてのサイドバーリンクが正しいページに遷移するか
  • 現在のページでサイドバー項目がハイライトされるか
  • ブラウザの戻る/進むが正しく機能するか

デザイン再現度を上げるコツ

スクリーンショットの品質

  • 解像度を高く -- Retina 2倍解像度で撮影
  • 全画面で撮る -- 部分ではなく画面全体を渡す
  • フルスクリーンショット -- DevToolsの "Capture full size screenshot" でスクロール全体

プロンプトの工夫

悪い例 # ❌ この画像を再現してください
良い例 # ✅ この画像は金融ダッシュボードのメイン画面です。 以下の点に特に注意して実装してください: - KPIカードのアイコンの背景色は各カードで異なる - テーブル偶数行に薄いグレー背景 - ステータスバッジは角丸のpillスタイル - 金額は3桁区切りのカンマフォーマット

デザインシステムの明示

フォント・カラー・角丸などをプロンプトで明示すると再現度が大幅に向上します。

プロンプトに含めるデザイン指示 デザインシステム: - フォント: Noto Sans JP - プライマリカラー: navy-900 (#102a43) - アクセントカラー: blue-500 (#3b82f6) - 成功: emerald-600 - 警告: amber-600 - エラー: red-600 - 角丸: xl (12px) カードに lg (8px) ボタンに使用

CLAUDE.md に書いておく

  • 技術スタック・ライブラリの指定
  • ファイル構成・命名規則
  • 使ってはいけないもの（NgModule等）
  • データフォーマットの規約
CLAUDE.md に一度書けば、毎回のプロンプトで繰り返す必要がなくなる

完成形のファイル構成

project/ ├── src/ │ ├── main.ts │ ├── index.html │ ├── styles.css # Tailwind imports │ └── app/ │ ├── app.component.ts # 共通レイアウト │ ├── app.routes.ts # ルーティング │ ├── pages/ │ │ ├── dashboard/ │ │ ├── portfolio/ │ │ ├── trade/ │ │ ├── risk/ │ │ ├── clients/ │ │ └── compliance/ │ ├── components/ │ │ ├── layout/ # Sidebar, Header │ │ ├── ui/ # KpiCard, Badge, Table │ │ └── charts/ # LineChart, PieChart │ ├── data/ # ダミーデータ │ ├── lib/ # ヘルパー関数 │ └── types/ # 共通型定義 ├── CLAUDE.md ├── angular.json └── package.json

ポイント

  • pages/ -- 各画面のページコンポーネント（6画面）
  • components/layout/ -- 全画面共通のSidebarとHeader
  • components/ui/ -- 再利用可能な汎用UIコンポーネント
  • components/charts/ -- Chart.js ベースのチャート群
  • data/ -- API未実装のためダミーデータを配置
  • types/ -- interface / type の共通定義
CLAUDE.md にこの構成を記載しておくことで、Claude Code が正しいディレクトリにファイルを生成する
