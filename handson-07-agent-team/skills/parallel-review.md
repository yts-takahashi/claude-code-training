# parallel-review スキル

複数の専門エージェントを並列起動して、コードを多角的にレビューするスキル。

## 使い方

「並列レビューして」「parallel-reviewを実行して」「コードを全方位でチェックして」と言われたら使う。

## 実行手順

1. ユーザーが指定したファイル（または現在のプロジェクト）を対象にする
2. セキュリティ・品質・ドキュメントの3エージェントを**同時に**起動する
3. 全エージェントの結果が揃ったら統合レポートを生成する

## プロンプトテンプレート

```
以下のファイルを3つの専門エージェントで並列レビューしてください。

対象ファイル: [FILE_PATH]

以下を同時に実行してください（並列実行）:
1. security-reviewer エージェントでセキュリティ問題を確認
2. code-quality-reviewer エージェントで品質・設計を確認  
3. doc-writer エージェントでドキュメントの不足を確認

全エージェントの結果が揃ったら、以下の形式で統合レポートを作成してください:

# コードレビューレポート: [FILE_PATH]

## 🔒 セキュリティ
[security-reviewerの結果]

## 📐 コード品質
[code-quality-reviewerの結果]

## 📝 ドキュメント
[doc-writerの結果]

## 総評
[3つの結果を踏まえた優先修正リスト]
```

## ポイント

- `同時に実行してください` と明示することで並列実行が促される
- 各エージェントは独立したコンテキストを持つため、互いの結果に影響されない
- 並列実行により直列の約3倍速でレビューが完了する
