# 演習3: 承認ワークフロー画面の作成

## 概要

融資申請の承認ワークフロー画面を作成してください。承認フローの進捗表示、承認・差戻しアクション、コメント入力を1つの画面で行える画面です。

## 使用コンポーネント

- `Timeline` - 承認フローのステップ表示
- `Modal` - 承認/差戻し確認ダイアログ
- `Button` - アクションボタン
- `StatusBadge` - 現在のステータス表示
- `FormField` - コメント入力、差戻し理由選択
- `Tabs` - 申請情報の表示切替
- `CurrencyDisplay` - 融資金額表示
- `AlertBanner` - 注意事項の表示
- `DataTable` - 過去の承認履歴表示

## 画面要件

### ファイル配置

```
src/pages/ApprovalWorkflow.tsx
```

### 画面構成

2カラムレイアウト（左: メインコンテンツ 2/3幅、右: サイドバー 1/3幅）

#### 案件ヘッダー

- 申請番号: `LA-2024-003`
- 申請者名: `鈴木一郎`
- 現在のステータス: `StatusBadge` で表示（processing）

`DataTable` で過去の承認アクションを表示:

| カラム | 備考 |
|---|---|
| アクション | 申請/承認/差戻し |
| 担当者 | |

#### アクションエリア

- コメント入力: `FormField` (textarea、プレースホルダー: 「審査コメントを入力してください...」)
- ボタン:
  - 「承認する」（primary）
  - 「差戻し」（danger）
  - 「保留にする」（ghost）

### 右カラム: サイドバー

#### 承認フロータイムライン

`Timeline` で以下のステップを表示:

1. **申請受付** - completed
   - 日時: 2024/03/10 09:00
   - 担当: 鈴木一郎（申請者）

2. **一次審査** - completed
   - 日時: 2024/03/15 14:30
   - 担当: 佐藤花子（審査担当）
   - 説明: 信用情報・財務諸表の審査完了

3. **担保評価** - current
   - 日時: 進行中
   - 担当: あなた
   - 説明: 担保評価の確認と承認

4. **二次審査（融資部長）** - upcoming
   - 担当: 高橋部長

5. **最終承認（支店長）** - upcoming
   - 担当: 山田支店長

6. **融資実行** - upcoming

#### 関連情報カード

- 同一顧客の既存融資件数: 3件
- 同一顧客の延滞履歴: なし
- 類似案件の平均承認日数: 12日

### モーダル

#### 承認確認モーダル

「承認する」ボタンクリック時に `Modal` (confirmMode) を表示:
- タイトル: 「融資承認の確認」
- 内容:
  - 申請番号: LA-2024-003
  - 融資金額: ¥100,000,000
  - 入力されたコメント
  - 「この操作は取り消せません。承認しますか？」
- 確認ボタン: 「承認する」（primary）
- キャンセルボタン: 「キャンセル」

#### 差戻しモーダル

「差戻し」ボタンクリック時に `Modal` を表示:
- タイトル: 「差戻しの確認」
- 内容:
  - `FormField` (select): 差戻し理由
    - 書類不備
    - 担保評価の再検討が必要
    - 追加資料の提出が必要
    - 融資条件の見直しが必要
    - その他
  - `FormField` (textarea): 詳細コメント（必須）
- 確認ボタン: 「差戻しする」（danger）
- キャンセルボタン: 「キャンセル」
