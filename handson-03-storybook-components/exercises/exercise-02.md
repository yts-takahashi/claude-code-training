# 演習2: 融資審査ダッシュボードの作成

## 概要

融資審査部門のマネージャーが使用するダッシュボード画面を、既存のFinUIコンポーネントを組み合わせて作成してください。KPI指標、チャート、案件一覧を1つの画面に集約します。

## 使用コンポーネント

- `KPICard` - 主要KPI指標の表示
- `LineChart` - 月次推移チャート
- `PieChart` - 構成比チャート
- `DataTable` - 最新案件一覧
- `AlertBanner` - 重要な通知表示
- `StatusBadge` - 案件ステータス表示
- `CurrencyDisplay` - 金額表示
- `Tabs` - ビュー切替

## 画面要件

### ファイル配置

```
src/pages/ReviewDashboard.tsx
```

### アラートエリア（上部）

ページ上部にアラートバナーを表示:
- 「5件の案件が承認期限を3日以内に迎えます」（warning タイプ、アクションボタン「確認する」付き）
- 閉じるボタン付き

### KPIエリア

4つの `KPICard` を横並びに配置（グリッドレイアウト: 4列）:

1. **融資残高**
   - 値: `¥12,345,678,900`
   - 前月比: `+3.2%`
   - スパークライン: 12ヶ月分のトレンドデータ

2. **新規申請件数**
   - 値: `142件`
   - 前月比: `+12.5%`
   - スパークライン: 12ヶ月分のトレンドデータ

3. **延滞率**
   - 値: `1.23%`
   - 前月比: `-0.15%`（改善を示す）
   - スパークライン: 12ヶ月分のトレンドデータ

4. **承認率**
   - 値: `87.5%`
   - 前月比: `+2.1%`
   - スパークライン: 12ヶ月分のトレンドデータ

### チャートエリア

`Tabs` で切り替え可能な2つのビュー:

#### タブ1: 月次推移

2カラムレイアウト:

- 左: `LineChart` - 融資残高推移（12ヶ月分）
  - 系列1: 新規融資額（青実線）
  - 系列2: 返済額（赤破線）
  - Y軸: 日本円フォーマット（億円単位）

- 右: `PieChart` - 審査結果の内訳（ドーナツチャート）
  - 承認（緑）
  - 審査中（黄）
  - 却下（赤）
  - 期限切れ（グレー）

#### タブ2: 業種別分析

2カラムレイアウト:

- 左: `PieChart` - 業種別融資残高（ドーナツチャート）
  - 製造業、サービス業、不動産業、小売業、IT・通信、その他

- 右: `LineChart` - 業種別延滞率推移
  - 系列ごとに業種（上位3業種）

### 案件一覧エリア

`DataTable` で直近の案件を表示:

| カラム | 備考 |
|---|---|
| 申請番号 | ソート可 |
| 申請者名 | ソート可 |
| 融資金額 | `CurrencyDisplay` で表示、ソート可 |
| ステータス | `StatusBadge` で表示 |
| 申請日 | ソート可 |
| 担当者 | |

- 表示件数: 5件
- ページネーション: なし
- ヘッダーに「すべて表示」リンク

### サンプルデータ

各チャートとKPIに以下のデータを用意:

```typescript
// 月次データ（12ヶ月分）
interface MonthlyData {
  month: string;         // '2024/04'
  loanBalance: number;   // 融資残高
  newLoans: number;      // 新規融資額
  repayments: number;    // 返済額
  applications: number;  // 申請件数
  approvalRate: number;  // 承認率(%)
  delinquencyRate: number; // 延滞率(%)
}

// 審査結果内訳
interface ReviewResult {
  name: string;  // '承認'
  value: number; // 245
  color: string; // '#16A34A'
}

// 業種別データ
interface IndustryData {
  name: string;  // '製造業'
  value: number; // 4500000000
}
```
