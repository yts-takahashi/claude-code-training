# 演習1: 融資申請一覧画面の作成

## 概要

融資申請の一覧を表示・管理する画面を、既存のFinUIコンポーネントを組み合わせて作成してください。

## 使用コンポーネント

- `DataTable` - 融資申請データの一覧表示
- `StatusBadge` - 各申請のステータス表示
- `SearchInput` - 申請番号・申請者名での検索
- `Button` - アクションボタン（新規申請、CSVエクスポートなど）
- `CurrencyDisplay` - 融資金額の表示

## 画面要件

### ファイル配置

```
src/pages/LoanApplicationList.tsx
```

### ヘッダーエリア

- 画面タイトル: 「融資申請一覧」
- 右側に以下のボタンを配置:
  - 「新規申請」ボタン（primary、左にプラスアイコン）
  - 「CSVエクスポート」ボタン（secondary、左にダウンロードアイコン）

### 検索・フィルターエリア

- `SearchInput` による検索（プレースホルダー: 「申請番号、申請者名で検索...」）
- ステータスフィルター用のボタングループ:
  - すべて / 審査中 / 承認済 / 却下 / 処理中 / 期限切れ
  - 選択中のフィルターはprimaryバリアント、それ以外はghostバリアント
- 検索はデバウンス300msで適用

### テーブルエリア

`DataTable` で以下のカラムを表示:

| カラム | キー | ソート | 備考 |
|---|---|---|---|
| 申請番号 | `id` | 可 | `LA-YYYY-NNN` 形式 |
| 申請者名 | `applicantName` | 可 | |
| 融資金額 | `amount` | 可 | `CurrencyDisplay` で表示 |
| 融資種別 | `loanType` | 可 | 事業資金/設備投資/運転資金/不動産担保 |
| ステータス | `status` | 可 | `StatusBadge` で表示 |
| 申請日 | `appliedDate` | 可 | YYYY/MM/DD 形式 |
| 支店 | `branch` | 可 | |

- ページネーション: 1ページ10件
- 行選択: 複数選択可能
- 行クリックで詳細画面へ遷移（console.logで代替可）

### サンプルデータ

以下のデータ構造で最低15件のサンプルデータを用意:

```typescript
interface LoanApplication {
  id: string;           // 'LA-2024-001'
  applicantName: string; // '田中太郎'
  amount: number;        // 50000000
  loanType: string;      // '事業資金'
  status: 'approved' | 'pending' | 'rejected' | 'processing' | 'expired';
  appliedDate: string;   // '2024-03-01'
  branch: string;        // '東京本店'
}
```

### 機能要件

1. **検索**: 申請番号または申請者名でフィルタリング（部分一致）
2. **ステータスフィルター**: 選択したステータスで絞り込み
3. **ソート**: カラムヘッダークリックで昇順/降順/初期状態を切替
4. **ページネーション**: 10件ずつ表示、ページ切替
5. **行選択**: 複数行選択、選択件数の表示
6. **一括操作**: 選択した行に対して「一括承認」ボタン（disabled: 選択なし時）
