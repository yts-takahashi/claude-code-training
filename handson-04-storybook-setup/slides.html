Storybook とは

概要

  • UIコンポーネントを独立して開発・テスト・ドキュメント化するツール
  • コンポーネントの全状態（バリアント）を一覧で確認できる
  • デザイナー・開発者間のコミュニケーションツールとして機能
  • Angular / React / Vue など主要フレームワークに対応

主な機能

Stories

コンポーネントの各状態を定義・表示

Controls

@Input をリアルタイムに操作

Actions

@Output イベントをログ表示

Docs

自動ドキュメント生成

Step 1: Claude Code で Storybook を導入

プロンプト例

このAngularプロジェクトに Storybook 8 を導入してください。 要件: - @storybook/angular を使用 - .storybook/ に設定ファイルを配置 - Tailwind CSS が適用されるよう設定 - storybook スクリプトを追加

検証チェックポイント

  • .storybook/main.ts が作成されているか
  • .storybook/preview.ts が作成されているか
  • package.json に依存関係が追加されているか
  • npm run storybook で起動するか
  • Tailwind CSS が Storybook 内で適用されているか
Storybookの初回起動には時間がかかる場合があります。エラーが出た場合はClaude Codeにエラーメッセージを貼り付けて修正を依頼してください。

Step 2: 最初のストーリーを作成 — Button

プロンプト例

src/app/components/Button/ button.component.ts を読んで、 Storybook ストーリーを作成してください。 - Default ストーリー - AllVariants（横並び表示） - AllSizes（横並び表示） - Loading / Disabled 状態 - Controls で全 @Input を操作可能に - title は 'Components/Button'

ストーリー構成

Default

primary / md のデフォルト状態

AllVariants

primary, secondary, danger, ghost を横並び

AllSizes

sm, md, lg を横並び

Loading

ローディングスピナー表示

Disabled

無効状態

Tip: Controls パネルで variant を切り替えて、全バリアントの色が正しいか確認しましょう

ストーリーファイルの基本構造

button.stories.ts import type { Meta, StoryObj } from '@storybook/angular'; import { ButtonComponent } from './button.component'; const meta: Meta<ButtonComponent> = { title: 'Components/Button', component: ButtonComponent, tags: ['autodocs'], argTypes: { variant: { control: 'select', options: ['primary', 'secondary', 'danger', 'ghost'], }, clicked: { action: 'clicked' }, }, }; export default meta; type Story = StoryObj<ButtonComponent>; export const Default: Story = { args: { variant: 'primary', size: 'md' }, };
Meta でコンポーネントの設定、StoryObj で個別のストーリーを定義する

Step 3: 残り5コンポーネントの一括生成

プロンプト例

残り5つのコンポーネント (Card, Badge, SearchInput, Avatar, ProgressBar) のストーリーを一括で作成してください。 Button のストーリーのフォーマットに 合わせてください。 各コンポーネントに Default + バリエーション ストーリーを含めてください。

Card

Default / Bordered / Hoverable / WithSubtitle

Badge

Default / AllVariants / AllSizes

SearchInput

Default + Actions表示

Avatar

Default / WithImage / AllSizes / Initials

ProgressBar

Default / AllColors / Complete / WithLabel

合計

6コンポーネント × 平均4ストーリー = 約24ストーリー

Step 4: ストーリーのカスタマイズ

Decorators

余白・背景色・レイアウトの設定

Play Functions

ユーザー操作のシミュレーション

Docs / JSDoc

自動ドキュメント生成

Global Settings

Step 5: コンポーネントとストーリーを同時作成

プロンプト例

新しい「Tooltip」コンポーネントを ストーリーと一緒に作成してください。 仕様: - selector: app-tooltip - @Input: text, position, delay - ホバーで表示/非表示 - Tailwind CSS / standalone ストーリー: - Default / AllPositions / WithDelay - Controls で操作可能

期待される成果物

src/app/components/Tooltip/ ├── tooltip.component.ts ← 新規 ├── tooltip.stories.ts ← 新規 └── index.ts ← 新規
コンポーネントとストーリーを同時に作ることで、最初からStorybookで確認しながら開発できる

ストーリー生成プロンプトのコツ

効果が低い

Storybookのストーリーを作って

どのコンポーネントか不明。最低限のストーリーしか生成されない

やや良い

Buttonのストーリーを作って。全バリアントを含めて

コンポーネント名はあるが、具体的な構成が不明

効果が高い

button.component.ts を読んで、 以下の構成でストーリーを作成: - Default: variant='primary' - AllVariants: 横並び表示 - Loading / Disabled 状態 Controls で全 @Input を操作可能に

コンポーネント実装を読ませ、具体的な構成を指示

Storybook 導入 5つのベストプラクティス

  1. Default + AllVariants を必ず作る
    最低限の2ストーリーで全状態を網羅
  2. Controls で全 @Input を操作可能にする
    手動テストの効率が劇的に向上
  3. Actions で @Output を確認する
    イベント発火の動作確認が容易に
  4. 実際のユースケースに近いデータを使う
    "Lorem ipsum" ではなく業務データ
  5. play function でインタラクションテストを追加
    手動確認の自動化

よくあるトラブルと対処法

Storybook が起動しない

キャッシュクリア:

rm -rf node_modules/.cache/storybook

診断コマンド:

npx storybook@latest doctor

Tailwind が適用されない

.storybook/preview.ts に styles.css をインポート:

import '../src/styles.css';

ストーリーが表示されない

.storybook/main.ts の stories パターンを確認

Meta の component が正しく指定されているか

@Output が Actions に表示されない

argTypes で action を定義しているか

addon-essentials がインストールされているか
